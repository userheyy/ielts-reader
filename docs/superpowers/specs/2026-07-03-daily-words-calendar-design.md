# 每日单词 + 日历记忆痕迹 — 设计文档

> 在记忆法词库/SRS 之上，加"每日任务"子系统：系统按记忆曲线每天派一组词（复习旧词+新学新词），日历热力图记录学习痕迹。背单词从"手动加入随机抽"升级为"每天清任务+看得见坚持"。
> 日期：2026-07-03　状态：头脑风暴逐段确认，直接进入实施（用户要求跳过实施计划）。

## 1. 背景与决策

用户反馈："加入生词库那种形式不如做个日历，随机抽40个一组，日历=记忆痕迹"。据此新增「每日」子系统。

| 决策点 | 选择 |
|---|---|
| 分组模式 | **B 动态每日任务**：不预先固定组；每天=到期复习词+补足的新词。日历格=当天学习痕迹 |
| 每日配额 | **C 用户可调**：默认档 20/30/40 + 自定义；出厂默认"标准 30 新词/天"。**复习不封顶(到期尽量全过)，只调新词量** |
| 布局 | 日历热力图(痕迹墙) 在上 + 今日任务(复习+新学两块) 在下 |
| 导航定位 | **A 新增「每日」tab 作主入口**；词库退回"浏览/检索"角色、生词库回归"个人生词"。三页职责:每日=背、词库=查、生词库=个人生词 |
| 数据存储 | **本地 localStorage**(纯静态不接API,无法存GitHub);纳入现有"全量备份/恢复"。不做跨设备自动同步 |

**关键澄清(用户问过)**:GitHub 仓库=只读的程序+内置词库,对所有人一样;用户个人数据(生词/打卡/进度)全在各自浏览器 localStorage,不回传 GitHub。同一网址在不同设备打开各有独立一份、互不相通。跨设备迁移靠手动"全量备份"导出文件再"恢复"。

## 2. 今日任务生成逻辑（核心，纯前端）

进入「每日」页时,若 `days[今天]` 不存在则生成并缓存当天(避免刷新换词);已存在则接着上次进度;跨天自然生成新一天。

1. **复习部分**:扫描所有已学词(内置词 `ielts_vocab_seed_review` + 生词 `ielts_vocab`),挑 `next_due ≤ 今天` 的,**全部**列入(受 `review_cap` 限,默认 null 不限)。
2. **新词部分**:从 `tools/seed_wordlist.json`(3575词按词频排序,已在项目内)按 `new_word_cursor` 往后取"还没学过且已在 vocab-seed.json 生成了 aids 的"词,补足到 `new_per_day`(默认30)。游标前移。
   - **约束**:新词只能取已在 `vocab-seed.json` 里有 aids 的词(当前36个);游标不会越过"已生成 aids 的边界"。未生成的词不放出(等后续批次)。
3. 今日任务 = 复习词 + 新词,复习优先排前。
4. 每词过一遍 = 复用现有 B 交互(词根提示→显示→三档评分)。评分复用现有 SRS:新词首次评分获得 `next_due`,将来自动回到某天复习。评分路由沿用 vocab.js 现有逻辑(生词走 store.gradeReview,内置词走 seed 独立 SRS)。

## 3. 数据结构

新增 localStorage 键 `ielts_daily`:
```jsonc
{
  "settings": { "new_per_day": 30, "review_cap": null },
  "new_word_cursor": 36,            // seed_wordlist 已放出到第几个高频词
  "days": {
    "2026-07-03": {
      "planned": 40, "reviewed_done": 14, "new_done": 12,
      "new_words": ["predict", "..."],   // 当天放出的新词(便于回看)
      "completed": false
    }
  }
}
```
- 热力图:某天格深浅 = 当天 `reviewed_done+new_done`;`completed` 满格亮。
- 连续打卡 = 从今天往回数连续有记录天数;累计词 = 各天 done 之和。
- 配额改动只影响当天(未取满则按新配额补)及之后,不追溯历史。
- 记忆状态仍在 `ielts_vocab` / `ielts_vocab_seed_review`;`ielts_daily` 只存任务编排+痕迹,职责不重叠。
